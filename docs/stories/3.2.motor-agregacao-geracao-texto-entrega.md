## Story
**As a** coordenador da CGTE,
**I want** que o sistema agrupe tarefas finalizadas por entrega PGD e gere texto descritivo,
**so that** o relatorio reflita fielmente o trabalho realizado no trimestre.

## Acceptance Criteria
1. Motor de agregacao: consulta tarefas finalizadas no periodo â†’ agrupa por categoria â†’ mapeia para entregas PGD via `taxonomy.json`
2. Para cada entrega, gera texto no formato: "[Quantidade] atividades realizadas: [lista dos titulos das tarefas]"
3. Exemplo de output: "5 atividades realizadas: Banner Dia das Maes, Banner Pascoa, Campanha eleitoral, Banner Dia Internacional da Mulher, Kit divulgacao processo seletivo"
4. Entregas com subtarefas incluem contagem de subtarefas finalizadas
5. Texto gerado respeita o padrao observado nos registros existentes do Petrvs
6. Testes unitarios para logica de agregacao com cenarios: trimestre cheio, trimestre vazio, categoria com multiplas entregas

## Dev Notes
- **Reporting Service:** [Source: architecture.md#5.5]
  - Responsabilidade: agregar tarefas por periodo/categoria/entrega e gerar texto PGD
  - Arquivo: `lib/report-generator.ts`
- **Report Flow:** [Source: architecture.md#7.2]
  ```
  Coordenador seleciona trimestre
  -> API Route busca tarefas finalizadas no periodo
  -> Report Generator agrupa por entrega PGD
  -> Texto por entrega gerado
  -> UI exibe relatorio estruturado
  ```
- **Taxonomy Integration:** [Source: architecture.md#5.4]
  - Usar `getEntregasPGD(categoriaId)` de `lib/taxonomy.ts` para resolver mapeamento
  - Uma categoria pode mapear para N entregas PGD (relacao N:N via taxonomy)
  - Uma entrega PGD pode receber tarefas de N categorias
- **Exemplo de mapeamento:** [Source: projeto-sistema-gestao-cgte.md#6.2]
  - Categoria "Comunicacao Visual" -> Entrega "Conteudo para Comunicacao Visual elaborado"
  - Categoria "Producao Audiovisual" -> Entregas "Material didatico docente gravado", "Material didatico editado/renderizado/publicado", "Video institucional do Cefor produzido"
- **Formato Petrvs:** Consultar registros existentes para manter compatibilidade. Provavel formato: narrativa simples listando atividades realizadas.
- **Subtarefas:** [Source: architecture.md#5.3]
  - `getAllSubtasks(taskId)` retorna subtarefas de uma tarefa
  - Filtrar por status finalizado
- **Kanboard timestamps:** `date_completed` e timestamp Unix. Converter para comparacao com periodo.
- **Depends on:** Story 3.1 (tela + API route base), Story 1.2 (kanboard-client), Story 1.3 (taxonomy)
- **File Locations:**
  - `lib/report-generator.ts` (novo)
  - Estender: `app/api/report-pgd/route.ts`
- **Performance:** Para relatorio trimestral, volume estimado ~50-100 tarefas por trimestre para equipe de 7. Nao e necessario paginacao.
