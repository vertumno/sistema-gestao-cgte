## Story
**As a** coordenador da CGTE,
**I want** um layout de dashboard profissional com navegacao clara,
**so that** eu consiga acessar rapidamente as diferentes visoes dos dados.

## Acceptance Criteria
1. Layout responsivo com sidebar de navegacao (desktop) e menu hamburger (mobile)
2. Navegacao com itens: Dashboard, Relatorio PGD, Configuracoes
3. Header com titulo "Sistema de Gestao CGTE" e identidade visual Cefor/Ifes
4. Componente de selecao de periodo (mes, bimestre, trimestre) com periodo atual pre-selecionado
5. Footer com versao do sistema e link para documentacao
6. Conformidade WCAG AA: contraste adequado, navegacao por teclado, aria-labels

## Dev Notes
- **Source Tree:** [Source: architecture.md#10, frontend-architecture.md#3]
  - Layout: `app/dashboard/layout.tsx`
  - Pagina: `app/dashboard/page.tsx`
  - Componentes layout: `components/layout/sidebar.tsx`, `header.tsx`, `footer.tsx`, `mobile-nav.tsx`
  - Componentes dashboard: `components/dashboard/period-selector.tsx`
  - Store: `stores/dashboard-store.ts`
  - Hook: `hooks/use-period-filter.ts`
- **State Management:** Zustand para filtros de periodo/area/servidor [Source: frontend-architecture.md#5.2]
- **Styling:** Tailwind + CSS variables com paleta Ifes (verde #0f766e, branco) [Source: frontend-architecture.md#8.2]
- **Component Pattern:** Seguir template de componente com `cn()` utility e tipos explÃ­citos [Source: frontend-architecture.md#4.1]
- **Naming:** PascalCase para componentes, kebab-case para rotas [Source: frontend-architecture.md#4.2]
- **shadcn/ui:** Usar componentes base (Button, Sheet para mobile nav, Select para periodo)
- **Routing:** App Router file-system based [Source: frontend-architecture.md#7.1]
  - `/dashboard` -> `app/dashboard/page.tsx`
  - `/relatorio-pgd` -> `app/relatorio-pgd/page.tsx`
  - `/configuracoes` -> `app/configuracoes/page.tsx`
- **WCAG AA:** Contraste minimo 4.5:1 para texto, 3:1 para texto grande. Focus visible obrigatorio. [Source: prd.md#3.4]
- **Previous Story Insights (1.4):** PoC implementada em `app/page.tsx` com layout basico institucional. O layout do dashboard deve evoluir esse visual mas em rota separada `/dashboard`.
- **Regra critica:** Nunca chamar Kanboard direto no client; usar apenas `/api/*` [Source: frontend-architecture.md#11.1]
- **Loading/Error/Empty:** Toda tela deve tratar os 3 estados [Source: frontend-architecture.md#11.1]
